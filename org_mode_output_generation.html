Motivation

Motivation

Output Generation

  • Generating output (texts, plots, and tables) is an essential part of research
  • Usually consists of small (~200 lines) scripts for each plot or table
  • Typically, you have to update these multiple times over the course of a Project
    • There is often a significant time interval between successive updates

Note test

Updating Output

  • You have to complete the following steps:
    1. Find the file(s) and the code blocks to generate the output (data, analysis, plot, and table)
    2. Re-learn the code so you know where to edit
    3. Run the code to update the plots

Challenges

  • You can easily end up with multiple files:

Challenges (continued)

  • Navigation becomes challenging if you use a long script
  • Re-learning is not easy due to poor annotation
  • If you are using one file, it is not always easy to run code in blocks
  • You have to edit non-Python (e.g., LaTeX) code
  • All these cause non-fluency

IPython Notebook

  • Better
    • Block-executing codes is easy
    • Annotation is nice and easy

IPython Notebook (continued)

  • However …
    • You have to run the IPython notebook kernel first to inspect a file
    • You have to navigate through your directory to find the notebook files
    • With multiple files, you have to open each notebook to inspect the code
    • Code navigation becomes clunky as your code becomes longer
    • Sometimes you have to edit non-Python (e.g., LaTeX) code

Motivation

  • Reduce overhead and minimize non-fluency in your workflow using org-mode

Org-mode

Org-mode

Org-mode

  • http://orgmode.org/
  • A major-mode in Emacs for keeping notes, maintaining TODO lists, planning projects, and authoring documents
  • Cross-platform
  • Convenient, flexible markup syntax for hierarchical documents
  • Execute and export code blocks and results
  • One tool for everything - the Python of text management
  • Flexible, can accommodate many different workflows
  • There are many useful features in org-mode, but in the interest of time, I'm going to focus only on features related to the output generation.
  • Last, org-mode is very flexible, and can accommodate many different workflows

Org-mode: Advantages

  • One hierarchical .org file for everything
    • No more managing multiple files
  • The file can contain heterogeneous contents:
    • TODO list, code (multiple languages), results, paper, everything
  • Excellent for:
    • Navigating
    • Annotating
    • Locating scripts
  • Powerful export facilities: beamer, LaTeX, html, reveal, etc.
  • Literate Programming functionalities (org-babel)

Conclusion

Takeaways

With org-mode, you can minimize overhead in output generation

  • Put everything about a project in one org file with a tree-like structure
  • Easy navigation of the trees
  • Good annotation capability
  • (Selectively) execute source code
  • Export source code or results
